# CLI コマンド設計ルール

この文書は、リポジトリ内の CLI コマンドを新規作成・拡張するときの共通ルールです。

## 用語

- ルートコマンド: 実行ファイル名に対応する最上位のコマンド。
- サブコマンド: ルートコマンドまたは別のサブコマンドの直下にあるコマンド。
- 現在の階層: 入力済みのコマンド列が示す階層。
- デフォルトサブコマンド: 明示的に省略可能と定義されたサブコマンド。
- 短縮オプション: `-h` や `-V` のような、長形式に対応する短いオプション。

## コマンド階層

コマンドは原則としてサブコマンド方式にします。

```text
<command> <subcommand> [nested-subcommand] [arguments] [options]
```

サブコマンドは処理の単位ごとに分けます。ルートコマンドと各サブコマンドは、それぞれの階層で利用できるサブコマンド、引数、オプション、ヘルプを持ちます。

## サブコマンド省略とデフォルト動作

サブコマンド省略時の動作は、コマンドごとに明示的に定義します。

- 引数もオプションもない場合の基本動作は、ルートヘルプの表示とする。
- デフォルトビューまたはデフォルトサブコマンドを定義した場合は、引数なしでそれを実行してよい。
- 入力や処理オプションがある場合、明確なデフォルトサブコマンドが定義されていれば、そのサブコマンドを補ってよい。
- デフォルト動作が定義されていない場合は推測して実行せず、ヘルプまたはエラーを表示する。
- サブコマンド省略記法と完全な記法は、同じ入力に対して同じ処理結果にする。
- 破壊的または曖昧な操作を暗黙のデフォルトにしない。

デフォルト動作と省略条件は、ルートヘルプとコマンドドキュメントに記載します。

## 複数型を受け付ける位置引数

1つの位置引数で複数の入力型を受け付ける場合は、明示指定と自動判定の両方を定義します。

```text
command [--type-a VALUE | --type-b VALUE | VALUE]
```

- 型ごとの明示フラグを用意する。
- 明示フラグ付きの入力は、指定された型として扱う。
- フラグなしの位置引数は、コマンドが定義した規則で自動判定する。
- 自動判定の優先順位と、判定できない場合の動作をヘルプとコマンドドキュメントに記載する。
- 同じ型・同じ入力であれば、明示指定と自動判定の処理結果を同じにする。
- 型が衝突する場合でも、明示フラグで目的の型を選択できるようにする。
- 明示フラグと、同じ入力を表す位置引数は原則として併用させない。
- 型を明示するためだけの形式的なフラグは、補完候補に表示しなくてよい。
- 補完候補に表示しない形式的なフラグも、ヘルプとコマンドドキュメントには必ず記載する。

実際のフラグ名、入力型、自動判定規則はコマンドごとに定義します。

## ヘルプ

ルートコマンドと各サブコマンドは、階層単位でヘルプを持ちます。

```sh
command help
command help subcommand
command --help
command subcommand --help
command subcommand nested-subcommand --help
```

- サブコマンド方式のコマンドは、ルート直下に `help` サブコマンドを実装する。
- `command help` はルートヘルプ、`command help subcommand [...]` は指定した階層のヘルプを表示する。
- ルートと各サブコマンドは、`-h` と `--help` の両方を受け付ける。
- ヘルプは現在の階層について説明する。
- 現在の階層に直下のサブコマンドがある場合は、全サブコマンドと各説明を記載する。
- 下位サブコマンドの引数・オプションの詳細は掲載せず、そのサブコマンドのヘルプに分ける。
- 現在の階層で利用できる全位置引数と全オプションを記載する。
- 補完候補に表示しない形式的なフラグも省略しない。
- 短縮オプションがある場合は、長形式と短縮形式の両方を記載する。
- 補完候補から短縮オプションを除外しても、ヘルプとコマンドドキュメントからは除外しない。
- サブコマンド省略時の動作を、その省略が可能な階層のヘルプに記載する。

引数なしでヘルプを表示するか、定義済みのデフォルト動作を実行するかは、コマンドごとに明記します。

## ルート共通コマンドとオプション

サブコマンド方式のコマンドは、ルート直下に `help` と `version` を実装し、対応するフラグ形式も実装します。

```sh
command help [subcommand...]
command version
command -h
command --help
command --online-help
command -V
command --version
```

- `help` と `version` はルートヘルプのサブコマンド一覧に記載する。
- `help`、`version`、および対応するフラグ形式は同じ結果を返す。
- `--online-help` は、共通の `internal/onlinehelp` を使用してコマンド名と `Version` から URL を生成する。
- バージョン表示は `-V` と `--version` の両方を受け付ける。
- `version`、`-V`、`--version`、`--online-help` はルート専用とし、操作サブコマンドの後ろでは受け付けない。
- サブコマンド固有の同等機能を追加する場合は、別の仕様として明記する。

## 補完

補完候補は現在の階層に応じて切り替えます。

- 現在の階層にサブコマンドがある場合は、直下のサブコマンドを候補に含める。
- 現在の階層で利用できるオプションを候補に含める。
- 下位サブコマンド専用のオプションは、サブコマンドが選択されるまで候補に含めない。
- デフォルトサブコマンドの処理オプションも、サブコマンドが選択されるまでは候補に含めない。
- サブコマンド選択後は、選択した階層に同じルールを適用する。
- `help` と `version` を補完上の正規形とし、対応する `-h` / `--help` と `-V` / `--version` は補完候補に含めない。
- 長形式がある短縮オプションは、補完候補に含めない。ただし、手入力した短縮オプションは受け付ける。
- 型を明示するためだけの形式的なフラグは、補完候補に含めなくてよい。
- オプション値の位置では、ファイル、ID、列挙値など、その値に適した候補を表示する。
- 候補をリスト表示する場合は、サブコマンドを上部、オプションを下部に表示する。
- Bash と Zsh で、同じ階層の候補と値補完を同じ仕様にする。

例:

```text
command <TAB>
  subcommand-a  -- description
  subcommand-b  -- description
  help          -- show help
  version       -- show version
  --online-help  -- show the versioned online help URL

command subcommand-a <TAB>
  --operation-option  -- description
```

## 実装・検証チェックリスト

- ルートと各サブコマンドのヘルプを実装する。
- `help` と `version` のサブコマンド、および対応するフラグ形式を実装する。
- 各階層のヘルプに、直下の全サブコマンド、全引数、全オプションを記載する。
- 長形式と短縮形式の両方をヘルプとコマンドドキュメントに記載する。
- サブコマンド省略とデフォルト動作をテストする。
- 明示的な入力型指定と自動判定が同じ結果になることをテストする。
- `help` / `--help` と `version` / `--version` が同じ結果になることをテストする。
- ルート専用のバージョン表示と `--online-help` が操作サブコマンド階層で拒否されることをテストする。
- 補完候補の階層、表示順、重複する help/version フラグと短縮オプションの除外、値補完を Bash と Zsh で確認する。
- Bash と Zsh の補完で、同じ位置引数を重複定義しない。
- 英語・日本語の `docs/commands/<command>.md` と `.ja.md` を更新する。
- 英語・日本語のリリースノートを更新する。
