# xhydra 1.0.0

`xhydra 1.0.0` runs Hydra against HTTP form authentication and stores successful credentials in ctx.

## Added

- Support raw HTTP request templates with `-r`/`--request`.
- Support direct URL and form-body input with `--url` and `--data`.
- Replace configured username and password form fields for Hydra requests.
- Support JSON failure matching with `--fail-json <field=value>`.
- Automatically select an installed password wordlist, with `-P`/`--password-list` for an explicit list.
- Save successful HTTP credentials to ctx.
- Save command output and execution status to ctx logs for both successful and failed runs.
