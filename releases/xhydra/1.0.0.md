# xhydra 1.0.0

`xhydra 1.0.0` runs Hydra against HTTP, SSH, FTP, and SMB authentication and stores successful credentials in ctx.

## Added

- Support raw HTTP request templates with `-r`/`--request`.
- Support direct URL and form-body input with `--url` and `--data`.
- Replace configured username and password form fields for Hydra requests.
- Support JSON failure matching with `--fail-json <field=value>`.
- Support SSH, FTP, and SMB modes with automatic target service selection and `--service`, `--host`, and `--port` overrides.
- Automatically select installed password wordlists, with `-P`/`--password-list` for an explicit list.
- Escalate across password wordlists up to the configured request limit while removing duplicate and previously searched passwords.
- Search usernames with a fixed password using `--password`, with automatic username wordlist selection or `-L`/`--user-list` for an explicit list.
- Track username search progress separately for each target, service, and fixed password.
- Show per-wordlist progress with `--status`, rerun the next automatic search with `--force`, and clear scoped progress with `--clear-cache`.
- Save successful HTTP, SSH, FTP, and SMB credentials to ctx.
- Save one command log per password wordlist with command output and execution status for both successful and failed runs.
- Provide Bash and Zsh completion for modes and options.
