# xssh 1.0.0

`xssh 1.0.0` は、ctx の現在のターゲットへ、保存済みの SSH credential と SSH service information を利用して接続します。

## 機能

- ctx Add-on として `xssh` CLI を提供します。
- 現在のターゲット、SSH credential、SSH service information を ctx JSON API から取得します。
- 保存済み username と password を利用して接続します。password は `sshpass -e` 経由で渡し、コマンド引数には含めません。
- password が保存されていない credential では通常の `ssh` を起動します。
- ctx に登録されていない username が指定された場合は、その username で通常の SSH 接続へフォールバックします。
- SSH service が保存されていない場合はポート22を使用します。
- 複数の credential または SSH service がある場合は対話的に選択できます。
- credential ID または username を指定して credential を選択できます。
- 最後に正常終了した credential を記憶し、次回の選択時にEnterを押すとデフォルトとして使用します。
- パスワードを含まない SSH コマンド、接続開始・終了時刻、状態、終了コード、stdout、stderr を ctx のログへ保存し、`xlog` で確認できます。

## 使い方

```sh
xssh
xssh <credential-id>
xssh <username>
```

`xssh` の実行には `ctx` と `openssh-client` が必要です。保存済み password を使用する場合だけ `sshpass` が必要です。

接続ログには password や `sshpass` の引数を保存しません。
