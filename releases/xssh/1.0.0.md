# xssh 1.0.0

`xssh 1.0.0` connects to the current ctx target using stored SSH credentials and saved SSH service information.

## Features

- Provides the `xssh` CLI as a ctx add-on.
- Reads the current target, SSH credentials, and SSH service information through the ctx JSON API.
- Connects with a stored username and password through `sshpass -e` without putting the password in command arguments.
- Starts normal `ssh` when the selected credential has no password.
- Falls back to normal SSH with a requested username when that username is not stored in ctx.
- Uses port 22 when no SSH service is stored.
- Selects among multiple stored credentials or SSH services interactively.
- Supports credential selection by credential ID or username.
- Remembers the last successfully used credential and uses it as the default when Enter is pressed at a later selection prompt.
- Records the sanitized SSH command, connection times, status, exit code, stdout, and stderr in ctx logs for review with `xlog`.

## Usage

```sh
xssh
xssh <credential-id>
xssh <username>
```

`xssh` requires `ctx` and `openssh-client`. `sshpass` is required only when a stored password is used.

Passwords and `sshpass` arguments are not included in connection logs.
