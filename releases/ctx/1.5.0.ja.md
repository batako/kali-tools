# ctx 1.5.0

`ctx 1.5.0`では、プラットフォームごとにprojectを分離できる名前付きproject rootを追加します。

## 追加

- 名前付きproject rootを登録する`ctx project root add <path> [--name <name>]`を追加しました。名前を省略した場合はパス末尾のディレクトリ名を使用します。
- activeなproject rootを切り替える`ctx project root use <name>`を追加しました。
- 登録済みrootとactiveなrootを確認する`ctx project root ls`を追加しました。
- ディレクトリやworkspaceデータを削除せず、inactiveなrootの登録だけを解除する`ctx project root rm <name>`を追加しました。
- ctx管理下のprojectを別の登録済みrootへ一括移動する`ctx project root move <from> <to>`を追加しました。workspace IDと収集済みデータを維持し、変更前の一括検証、`--dry-run`、失敗時のファイルシステムロールバックに対応します。
- 登録済みproject root名のBash/Zsh補完を追加しました。
- 探索ツールが収集したWeb pathを表示する`ctx web ls`と`xweb`シェルショートカットを追加しました。Target選択とバージョン付きJSON出力に対応します。
- 明示的な確認後に、プライマリまたは選択したTargetの収集済みWeb path、xgobusterのwordlist履歴、検索済みwordキャッシュを一括削除する`xweb clear`を追加しました。コマンドログと無関係な探索キャッシュは保持します。
- Web探索結果に`path`、`param-name`、`param-value`の種別とparameter詳細を保存できるようにしました。
- `xweb --type <type>`による絞り込みと、`xweb show <ID>`による詳細表示を追加しました。
- `xweb`のBash/Zsh補完で、`--type`と`--format`に指定可能な値を候補表示します。
- 引数なしの`xweb`と`ctx web`でも、既定の`ls`で利用できるオプションを補完します。

## 互換性

- 既存の単一`project.root`設定は維持され、`default`rootとして扱われます。
- `ctx project root [path]`と`project.root`設定キーは、引き続きactive rootのパスを表します。
- 既存のWeb探索結果は`path`として扱われます。
