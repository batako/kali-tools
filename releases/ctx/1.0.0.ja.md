# ctx 1.0.0

ctx 1.0.0 は、ターゲット中心の作業を整理するためのCLIツールです。

現在のworkspace、target、hostname、scan結果、credential、note、command履歴をまとめて管理できます。

## ハイライト

- 調査や作業ごとにworkspaceを作成・管理できます。
- primary target IPと名前付きtargetを管理できます。
- hostnameを保存し、ctx管理の `/etc/hosts` entryを同期できます。
- `ctx x` でコマンドを実行し、stdout、stderr、exit status、timestampを保存できます。
- noteを追加し、`ctx log` でcommand logと合わせたtimelineを確認できます。
- `ctx scan` でnmap scanを実行し、`ctx service` で保存済みservice情報を確認できます。
- `ssh` や `wordpress` などのscope付きcredentialを保存できます。
- `ctx prompt` でshell prompt向けの情報を出力できます。
- prompt、credential、service、formatsの情報をJSONで取得できます。
- `ctx init-shell` でshell completionとhelper commandを導入できます。

## よく使うコマンド

```bash
ctx workspace init
ctx target set 10.10.10.10
ctx host add target.local
ctx scan -p 22,80,443
ctx service ls
ctx credential set ssh root toor
ctx note "interesting finding"
ctx log
ctx prompt
ctx x whoami
```

## Shell Integration

`ctx init-shell` 実行後は、`xinit`、`xtarget`、`xscan`、`xservice`、`xcredential`、`xnote`、`xlog`、`xprompt`、`x` などのhelper commandを使えます。

optional extra shortcutとして `pj`、`ta`、`cr` も利用できます。

## JSON出力

ctx 1.0.0 は、外部ツールやAdd-on向けにバージョン管理されたJSON出力を提供します。

利用できるJSON出力:

- `formats`
- `prompt`
- `credential`
- `service`

## Maintenance

- `ctx doctor` でctxの設定状態を確認し、修正案を表示できます。
- `ctx reset` でctx本体をアンインストールせず、ctx管理データとshell integrationを削除できます。
- amd64 / arm64向けのDebian packageを提供します。
