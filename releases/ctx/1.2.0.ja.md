# ctx 1.2.0

ctx 1.2.0 は、ctx 1.1.0 からのアップグレードです。設定・プロジェクト管理を改善し、ワークスペース識別子の保存方法を変更し、Add-on 向けのコマンドログAPIを追加しました。

## 新機能

- ctx の設定を管理する `ctx config` と、シェルショートカット `xconfig` を追加しました。
- `project.root` 設定を操作する `ctx config get`、`ctx config set`、`ctx config ls` を追加しました。
- `ctx project ls` がプロジェクトIDと名前を表示するようになりました。
- `ctx project rm` でプロジェクトIDまたは名前を指定できるようになりました。
- `ctx workspace ls` がヘッダー付きでワークスペースのID、名前、パスを表示するようになりました。
- Add-on が ctx のデータベースへ直接アクセスせずにコマンドログを開始・完了できる、`ctx log start` と `ctx log finish` のJSONエンドポイントを追加しました。

## ワークスペースとデータベースの変更

- データベース内のワークスペースレコードが内部整数IDを使用するようになりました。
- ワークスペース名とルートパスを分離して保存し、ルートパスは引き続き一意に管理します。
- 既存のワークスペースデータディレクトリは、これまでどおりUUIDベースのディレクトリ名を使用します。
- ターゲット、credential、service、note、command log などの関連レコードも、新しいワークスペースIDへ移行されます。

## アップグレードについて

- 既存の ctx 1.1.0 のデータベースは、ctx 起動時に自動でマイグレーションされます。
- 手動でマイグレーションコマンドを実行する必要はありません。
- マイグレーション中も既存のワークスペースデータディレクトリは維持されます。
- `pj root`、`pj <name>`、`pj rm <name>` など、既存の日常的な操作は引き続き利用できます。

## ユーザー向け

- 設定されているプロジェクトルートは `ctx config get project.root` で確認できます。
- プロジェクトルートを変更するには `ctx config set project.root <path>` を使用します。
- プロジェクトを削除する前に `ctx project ls` でプロジェクトIDを確認してください。
- 名前だけでは対象を一意に特定できない可能性がある場合は、`ctx project rm <id>` を使用してください。

## Add-on 向け

Add-on は ctx のデータベースへ直接アクセスせず、バージョン管理されたJSONログAPIを利用できます。

```sh
ctx log start --format json --format-version 1
ctx log finish <id> --format json --format-version 1
```

開始・完了リクエストの本文は標準入力から読み込み、レスポンスはJSONとして標準出力へ出力します。
