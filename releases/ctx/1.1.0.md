# ctx 1.1.0

ctx 1.1.0 improves upgrade safety by adding automatic database migrations.

This release does not change the main day-to-day commands. It prepares ctx for future updates so existing workspaces can keep working as the database grows over time.

## What's New

- ctx can now upgrade its database automatically when the schema changes.
- Existing ctx 1.0.0 databases are recognized and prepared for future migrations.
- New databases are created with the latest supported schema.
- Release checks now include migration tests using a real ctx 1.0.0 database fixture.

## For Users

- You do not need to run a manual migration command.
- Existing workspaces should continue to open normally after upgrading.
- If ctx detects an unknown or damaged database schema, it will stop instead of guessing and risking data corruption.

## Notes

- This release is mainly a compatibility and reliability update.
- The visible CLI behavior is intended to remain the same as ctx 1.0.0.
