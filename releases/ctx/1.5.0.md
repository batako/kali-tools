# ctx 1.5.0

`ctx 1.5.0` adds named project roots for keeping projects from different platforms separate.

## Added

- Added `ctx project root add <path> [--name <name>]` to register named project roots, deriving the name from the final path component by default.
- Added `ctx project root use <name>` to switch the active project root.
- Added `ctx project root ls` to list registered roots and show the active selection.
- Added `ctx project root rm <name>` to unregister an inactive root without deleting its directory or workspace data.
- Added `ctx project root move <from> <to>` to move all ctx-managed projects to another registered root while preserving workspace IDs and collected data. The command validates the complete move before changing anything, supports `--dry-run`, and rolls back filesystem changes if the move fails.
- Added Bash and Zsh completion for registered project root names.
- Added `ctx web ls` and the `xweb` shell shortcut for viewing web paths collected by discovery tools, with target selection and versioned JSON output.
- Added `xweb clear` to remove collected web paths, xgobuster wordlist history, and cached searched words for the primary or selected target after explicit confirmation. Command logs and unrelated discovery caches are preserved.
- Added structured `path`, `param-name`, and `param-value` web discoveries.
- Added `xweb --type <type>` filtering and `xweb show <ID>` details.
- Added Bash and Zsh value completion for the `xweb --type` and `xweb --format` options.
- Added default `ls` option completion when using `xweb` or `ctx web` without an explicit subcommand.
- Added `ctx wordlist [ls]|show` to inventory all files available in the runtime wordlists root and recommend usable lists by task. `ctx wordlist` and `ctx wordlist ls` behave identically, and `--kind` uses recommendation order. The required `wordlists` package and optional packages or broken links are reported with availability state.
- Embedded a known catalog containing SHA-256, format, line count, and multiple per-kind assessments for every wordlist in the recommended Kali environment. Unknown files are conservatively classified from small samples, while verified known files are ranked first using per-kind priorities.
- Added a shared recommendation API that gives bundled tools the same candidates and order as `ctx wordlist --kind <kind>`, making ctx the single automatic-selection source for xffuf, xgobuster, and xhydra.
- Added `ctx wordlist extract` to safely extract only `rockyou.txt.gz` versions matching the trusted SHA-256 embedded in ctx, with confirmation bypass, output replacement, and post-success source removal options.

## Compatibility

- Existing single `project.root` settings are preserved and treated as the `default` root.
- `ctx project root [path]` and the `project.root` configuration key continue to represent the active root path.
- Existing web discoveries are treated as `path` discoveries.
