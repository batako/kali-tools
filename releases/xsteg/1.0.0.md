# xsteg 1.0.0

`xsteg 1.0.0` is the first release of a utility that detects hidden data in local files, cracks supported passphrases, and extracts payloads.

## Added

- Added recursive file and directory processing through `scan` and `extract`.
- Added common analysis with file, ExifTool, Binwalk, and strings.
- Added StegSeek cracking and extraction for steghide payloads in JPEG, BMP, WAV, and AU files.
- Tried password wordlists in the order selected by the `ctx wordlist` password kind, with `--wordlist` override and `--no-crack` support.
- Separated detection-only `scan` from extraction-only `extract`. Extraction requires and reuses a same-SHA-256 scan result, and instructs users to run `scan` first when none exists.
- Prompted for automatic StegHide analysis, a known passphrase, or skipping, with `--auto` and `--manual` preselection.
- Kept manually entered passphrases hidden and preserved sanitized original embedded filenames.
- Returned `FAILED` and a non-zero exit status for rejected manual passphrases, reserving `EXTRACTED` for successful extraction.
- Displayed progress while `scan` performs StegHide seed detection.
- Added optional zsteg support for PNG/BMP and stegsnow support for text files.
- Restricted Binwalk extraction to non-executing carve mode and limited extracted file count and size.
- Stored source SHA-256, backend runs, attempted wordlists, and findings in a mode-`0600` `report.json`.
- Made the no-argument command list saved reports and added `show` for report details.
- Added `doctor` plus Bash and Zsh completion.

## Examples

```sh
xsteg scan image.png
xsteg extract image.jpg
xsteg extract image.jpg --manual
xsteg extract image.jpg --wordlist ./passwords.txt
xsteg ls
xsteg show 1
xsteg doctor
```

Extracted data is treated as untrusted and is never executed automatically. Required functionality remains available when the optional zsteg and stegsnow backends are absent.
