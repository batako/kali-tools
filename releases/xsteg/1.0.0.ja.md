# xsteg 1.0.0

`xsteg 1.0.0`は、ローカルファイル内の隠しデータを検出し、必要に応じてパスワードを解析して抽出する初回リリースです。

## 追加

- ファイルまたはディレクトリを再帰走査する`scan`と`extract`を追加しました。
- `file`、ExifTool、Binwalk、stringsを使った共通分析を追加しました。
- JPEG、BMP、WAV、AUのsteghideペイロードをStegSeekで解析・抽出します。
- StegSeekのワードリストは、`ctx wordlist --kind password`相当の選定結果を優先順に使用します。`--wordlist`による上書きと`--no-crack`にも対応しました。
- `scan`を検出専用、`extract`を抽出専用に分離しました。`extract`は同一SHA-256のscan結果を必須とし、scan未実行時は先にscanするよう案内します。
- StegHide抽出では自動解析、手動入力、スキップから選択し、`--auto`と`--manual`で方式を事前指定できます。
- 手動パスワードは端末へ表示せず、埋め込み元のファイル名を安全な名前に正規化して維持します。
- 誤った手動パスワードは`FAILED`と終了コード1を返し、抽出成功時だけ`EXTRACTED`を表示します。
- `scan`のStegHide seed検査中に進捗を表示します。
- PNG/BMP向けのzsteg、テキスト向けのstegsnowを任意バックエンドとして利用します。
- Binwalkは外部抽出コマンドを実行しないcarveモードに限定し、抽出数とファイルサイズを制限しました。
- 元ファイルのSHA-256、実行ツール、試行済みワードリスト、発見内容を権限`0600`の`report.json`へ保存します。
- 引数なしで保存済み結果を一覧表示し、`show`で詳細を確認できます。
- `doctor`とBash/Zsh補完を追加しました。

## 使用例

```sh
xsteg scan image.png
xsteg extract image.jpg
xsteg extract image.jpg --manual
xsteg extract image.jpg --wordlist ./passwords.txt
xsteg ls
xsteg show 1
xsteg doctor
```

解析対象は信頼できないデータとして扱い、抽出物は自動実行しません。zstegとstegsnowが未導入でも必須機能は利用できます。
