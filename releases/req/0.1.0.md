# req 0.1.0

req 0.1.0 is a small CLI for replaying raw HTTP requests saved in `.req` files.

It is useful when you want to save a request from a proxy, editor, or note and send it again from the terminal.

## Highlights

- Send a raw HTTP request from a `.req` file.
- Preserve request headers and body.
- Support common HTTP methods such as `GET`, `POST`, `PUT`, `DELETE`, `PATCH`, `OPTIONS`, and `HEAD`.
- Resolve the target URL from an absolute request URL or from the `Host` header.
- Infer HTTP or HTTPS from request headers when possible.
- Force HTTPS with `-S` or `--https`.
- Print the HTTP response to standard output.

## Usage

```bash
req request.req
req --https request.req
```

## Notes

- Redirects are not followed automatically.
- `req` is distributed as its own Debian package.
