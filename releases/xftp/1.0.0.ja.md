# xftp 1.0.0

`xftp 1.0.0` は、ctx が提供するFTP credentialとFTP service informationを利用して、現在のctxターゲットへ接続します。

## 機能

- ctx Add-on として `xftp` CLI を提供します。
- 現在のターゲット、FTP credential、FTP service informationをctx JSON APIから取得します。
- 選択したusernameとFTP portを使い、`lftp` 経由で接続します。
- 保存済みpasswordはコマンド引数に含めず、`LFTP_PASSWORD` と `lftp --env-password` を利用して渡します。
- 接続開始時に保存済みpasswordの認証を確認し、認証に失敗した場合は未認証のコンソールを開かずに終了します。
- credentialを使う接続では、無制限の再接続を無効にします。
- FTP credentialが保存されていない場合は、通常のFTP接続へフォールバックします。
- 指定したusernameが登録されていない場合は、そのusernameを使った通常のFTP接続へフォールバックします。
- FTP serviceが保存されていない場合はポート21を使用します。
- 複数のcredentialまたはFTP serviceがある場合は対話的に選択できます。
- 最後に正常終了したcredentialを記憶し、次回のcredential選択時にEnterを押すとデフォルトとして使用します。
- credential IDまたはusernameを指定してcredentialを選択できます。
- passwordを含まないFTPコマンド、接続開始・終了時刻、状態、終了コード、stdout、stderrをctxのログへ保存し、`xlog`で確認できます。
- 接続ログにはpasswordや`LFTP_PASSWORD`の値を保存しません。

## 使い方

```sh
xftp
xftp <credential-id>
xftp <username>
```

`xftp` の実行には `ctx` と `lftp` が必要です。

## ログ

各接続ではctxのcommand-log APIを利用します。接続の開始・終了と`lftp`が出力した内容を記録し、passwordと認証用の環境変数は除外します。
