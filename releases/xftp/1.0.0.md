# xftp 1.0.0

`xftp 1.0.0` connects to the current ctx target using FTP credentials and FTP service information provided by ctx.

## Features

- Provides the `xftp` CLI as a ctx add-on.
- Reads the current target, FTP credentials, and FTP service information through the ctx JSON API.
- Connects through `lftp` using the selected username and FTP port.
- Passes stored passwords through `LFTP_PASSWORD` and `lftp --env-password` instead of command arguments.
- Verifies stored-password authentication when the connection starts and exits on authentication failure instead of opening an unauthenticated console.
- Disables unlimited connection retries for credential-based connections.
- Falls back to normal FTP when no FTP credential is stored.
- Falls back to a normal username-based FTP connection when the requested username is not registered.
- Uses port 21 when no FTP service is stored.
- Selects among multiple stored credentials or FTP services interactively.
- Remembers the last successfully used credential and uses it as the default when Enter is pressed at the next credential selection prompt.
- Supports credential selection by credential ID or username.
- Records the sanitized FTP command, connection times, status, exit code, stdout, and stderr in ctx logs for review with `xlog`.
- Does not record passwords or `LFTP_PASSWORD` values in connection logs.

## Usage

```sh
xftp
xftp <credential-id>
xftp <username>
```

`xftp` requires `ctx` and `lftp`.

## Logging

Each connection uses the ctx command-log API. The log includes the connection lifecycle and the output produced by `lftp`, while passwords and authentication environment variables are excluded.
