# xgobuster 1.3.0

`xgobuster 1.3.0` supports continuing web discovery while filtering wildcard responses.

## Added

- Added `--exclude-status <code>` to exclude HTTP status codes. Multiple codes can be separated by commas, and the default `404` exclusion remains active.
- Added Bash and Zsh completion for `--exclude-status`.

## Changed

- Included `--exclude-status` and `--exclude-length` in search history conditions so runs with different response filters are tracked separately.

## Removed

- Removed `xgo --sitemap` and `xgobuster --sitemap`. Use the dedicated `xweb` viewer for collected web paths.
