# xgobuster 1.0.0

`xgobuster 1.0.0` runs Gobuster against the current ctx target and stores web discovery results in ctx.

## Features

- Provides the `xgobuster` CLI and the short `xgo` command.
- Resolves the target URL from ctx web services, with support for HTTP and HTTPS.
- Automatically selects web-content wordlists from the Kali wordlist directory.
- Supports the `web-quick`, `web-standard`, and `web-deep` search profiles.
- Supports technology presets for file searches, including PHP, WordPress, ASP.NET, Java, Node.js, and static sites.
- Accepts explicit Gobuster extensions with `-x` or `--extensions`.
- Shares extensionless path history between directory and file searches while tracking extension requests separately.
- Continues searches within the configured request limits with `--next` and supports explicit reruns with `--force`.
- Shows wordlist and search progress with `--status`.
- Supports explicit wordlists with `-w` or `--wordlist`.
- Saves Gobuster command output, status, exit code, and discovered paths through ctx.
- Provides Bash and Zsh completion for `xgobuster` and `xgo`.

## Usage

```sh
xgo
xgo --next
xgo --preset wordpress
xgo -x php,html,js
xgo --status
xgo --status --preset wordpress
```

`xgobuster` requires `ctx`, `gobuster`, and the Kali `wordlists` package.

## Search state

Extensionless paths are shared with directory searches because Gobuster checks the base path when `-x` is used. Extension-specific requests are tracked independently so a directory search cannot incorrectly mark a WordPress or other technology-specific file search as complete.
