# xdec 1.0.0

## Overview

Added `xdec`, a Go-based frontend that unifies literal values, files, and stdin for instant decoding and password recovery. John and hashcat are selected internally as recovery backends.

## Highlights

- Literal, `-f` file, and stdin input with candidate extraction
- Positional detection of existing regular files, so `-f` can be omitted
- Built-in Base64 and hex decoding
- Classification for MD5, NTLM, MD4, SHA-1, SHA-256, bcrypt, and Argon2 prefixes
- Mandatory confirmation before expensive recovery and `--yes` for non-interactive input
- Automatic ctx password-wordlist selection and sequential multi-wordlist recovery
- xdec parent logs with one xlog child step per wordlist
- Automatic SSH private-key detection, encryption-state checking, and `ssh2john` integration
- Immediate handling of unencrypted SSH keys without invoking John
- Username extraction and automatic ctx credential saving when username and scope are known
- Per-input state, completed-attempt skipping, and display of cached recovered values
- `--refresh` to re-analyze only the current input
- Human-readable and JSON output plus bash and zsh completions
- `--online-help` to print the versioned online documentation URL
- `decode`, `help`, and `version` subcommands plus flag forms for help and version output
- Explicit `--file` / `--string` input-type selection overriding automatic detection

## Security

Recovered values are stored in state, which is written with mode 0600. Private-key contents and recovered passphrases are not written to xlog.
