# xsmb 1.0.0

`xsmb 1.0.0` は、ctx が提供するSMB credentialとSMB service informationを利用して、現在のctxターゲットで選択したSMB共有へ接続します。

## 機能

- ctx Add-on として `xsmb` CLI を提供します。
- 現在のターゲット、SMB credential、SMB service informationをctx JSON APIから取得します。
- `smbclient -L` でディスク共有を取得し、選択した共有へ `smbclient` で接続します。
- 特殊共有である `IPC$` は選択肢から除外します。
- 保存済みpasswordはコマンド引数に含めず、`PASSWD` 環境変数を利用して渡します。
- SMB credentialが保存されていない場合は、通常のSMB共有一覧取得へフォールバックします。
- 指定したusernameが登録されていない場合は、そのusernameを使った通常のSMB接続へフォールバックします。
- SMB serviceが保存されていない場合はポート445を使用します。
- `smb`、`microsoft-ds`、`netbios-ssn` のservice名に対応します。
- 複数のcredentialまたはSMB serviceがある場合は対話的に選択できます。
- 最後に正常終了したcredentialを記憶し、次回のcredential選択時にEnterを押すとデフォルトとして使用します。
- credential IDまたはusernameを指定してcredentialを選択できます。
- passwordを含まないSMBコマンド、接続開始・終了時刻、状態、終了コード、stdout、stderrをctxのログへ保存し、`xlog`で確認できます。
- 接続ログにはpasswordや`PASSWD`の値を保存しません。

## 使い方

```sh
xsmb
xsmb <credential-id>
xsmb <username>
```

`xsmb` の実行には `ctx` と `smbclient` が必要です。
