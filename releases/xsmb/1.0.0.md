# xsmb 1.0.0

`xsmb 1.0.0` connects to a selected SMB share on the current ctx target using SMB credentials and service information provided by ctx.

## Features

- Provides the `xsmb` CLI as a ctx add-on.
- Reads the current target, SMB credentials, and SMB service information through the ctx JSON API.
- Discovers disk shares through `smbclient -L` and connects to the selected share with `smbclient`.
- Excludes the special `IPC$` share from the selectable shares.
- Passes stored passwords through the `PASSWD` environment variable instead of command arguments.
- Falls back to normal SMB share listing when no SMB credential is stored.
- Falls back to a normal username-based SMB connection when the requested username is not registered.
- Uses port 445 when no SMB service is stored.
- Recognizes `smb`, `microsoft-ds`, and `netbios-ssn` service names.
- Selects among multiple stored credentials or SMB services interactively.
- Remembers the last successfully used credential and uses it as the default when Enter is pressed at the next credential selection prompt.
- Supports credential selection by credential ID or username.
- Records the sanitized SMB command, connection times, status, exit code, stdout, and stderr in ctx logs for review with `xlog`.
- Does not record passwords or `PASSWD` values in connection logs.

## Usage

```sh
xsmb
xsmb <credential-id>
xsmb <username>
```

`xsmb` requires `ctx` and `smbclient`.
